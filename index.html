TryHackMe · Web Security
SQL Injection — Authentication Bypass

Exploit a login form vulnerable to classic SQLi to gain admin access without valid credentials.

Difficulty

Problem Statement

A vulnerable web app exposes a login endpoint where user-supplied input is concatenated directly into an SQL query. The goal is to authenticate as the admin without knowing the password.

Approach
  • Identified the login form and inspected the POST request via browser DevTools.
  • Tested with ' OR '1'='1 as the username to detect SQL syntax errors.
  • Used the payload admin'-- which comments out the password check in the query.
  • Confirmed bypass with a 302 redirect to the admin dashboard.
  • Used SQLMap to enumerate the database structure and extract the users table.
Tools Used
Burp Suite SQLMap Firefox DevTools
Screenshot
[ SQLMap output & admin dashboard screenshot ]
Key Lessons
  • Parameterised queries / prepared statements completely prevent SQLi.
  • Input validation alone is insufficient — defence must be at the query layer.
  • SQLMap's --level and --risk flags are essential for thorough enumeration.
HackTheBox · Forensics
Memory Forensics — Malware Analysis

Analyse a Windows memory dump to identify a running malicious process and extract indicators of compromise.

Difficulty

Problem Statement

A Windows 10 memory image was captured from a compromised machine. Identify the malicious process, extract the C2 IP address, and recover the dropped payload name.

Approach
  • Ran volatility -f mem.dmp imageinfo to confirm profile as Win10x64_19041.
  • Listed processes with pslist and pstree — spotted an unusual svchost.exe with no parent PID.
  • Used netscan to reveal an established connection to 198.51.100.42:4444.
  • Dumped the process memory with procdump and uploaded to VirusTotal — detected as Meterpreter reverse shell.
  • Extracted dropped file path from process handles using handles -p <PID>.
Tools Used
Volatility 3 VirusTotal strings CyberChef
Screenshot
[ Volatility pslist & netscan output ]
Key Lessons
  • Memory forensics can recover artefacts that are wiped from disk.
  • Meterpreter is fully in-memory — disk signatures are often absent.
  • Process parent-child relationships are crucial for spotting anomalies.
PicoCTF · Cryptography
RSA Weak Keys — Factorisation Attack

Break an RSA-encrypted message where the modulus N was generated with small prime factors.

Difficulty

Problem Statement

Given: public key (n, e) and ciphertext c. The RSA modulus n was created by multiplying two small primes. Recover the plaintext flag.

Approach
  • Used factordb.com to factor n instantly — returned p and q.
  • Computed Euler's totient: φ(n) = (p−1)(q−1).
  • Calculated the private key d = e⁻¹ mod φ(n) using the extended Euclidean algorithm in Python.
  • Decrypted the ciphertext: m = c^d mod n, then converted from integer to ASCII.
Tools Used
Python (sympy) factordb.com CyberChef
Screenshot
[ Python decryption script output ]
Key Lessons
  • RSA security fundamentally depends on the difficulty of factoring large n — small primes are trivially factorable.
  • Real-world RSA uses 2048-bit or 4096-bit keys with cryptographically random prime generation.
  • Always use well-audited cryptographic libraries rather than rolling your own RSA.
